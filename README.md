# Aurora
Локальный медиаплеер с интерфейсом Liquid Glass, объединяющий музыку, радио, IPTV и аудиокниги.

- **Моя музыка** — локальная библиотека (треки + плейлисты)
- **Радио** — интернет-радио (aacp / AAC+ / mp3)
- **ТВ** — IPTV по M3U-плейлистам (HLS / MPEG-TS / DASH)
- **Аудиокниги** — папка или ZIP с mp3-главами, скорость воспроизведения, навигация по главам

Переключение режимов не прерывает воспроизведение — Player Bar внизу общий.

## Стек

- React 18 + Vite + TypeScript
- Tailwind CSS (тёмная тема, акцент `#1DB954`)
- Zustand — единый стор плеера
- IndexedDB (`idb-keyval`) — станции, плейлисты, file-handles, настройки
- `music-metadata` — теги и обложки треков
- `hls.js` — HLS-потоки
- `mpegts.js` — сырые MPEG-TS потоки
- `jszip` — распаковка zip-архивов с аудиокнигами

Никакого backend и БД — всё хранится в браузере.

## Запуск через Docker (рекомендуется)

Требуется только Docker + Docker Compose. Node.js и npm на машине не нужны.

```bash
docker compose up -d --build
```

Откройте <http://localhost:8080>.

Остановить:

```bash
docker compose down
```

Multi-stage Dockerfile собирает приложение внутри контейнера (Node 20-alpine), затем раздаёт статику через nginx — итоговый образ ~50 MB.


## Запуск локально (без Docker)

Нужен Node.js ≥ 20.

```bash
npm install
npm run dev      # dev-режим → http://localhost:5173
npm run build    # production-бандл в dist/
npm run preview  # локальный просмотр продакшен-бандла
```

## Использование

### Моя музыка

Две вкладки: **Треки** и **Плейлисты**.

**Добавление музыки:**

- **«Добавить папку»** (Chrome / Edge / Opera) — выбираете корневую папку через File System Access API. Приложение рекурсивно сканирует её, читает теги и обложки. Permission запоминается в IndexedDB — при следующем запуске нужно лишь подтвердить доступ, выбирать папку заново не придётся.
- **«Добавить файлы»** — выбор отдельных файлов; работает во всех браузерах (для Firefox / Safari это единственный путь).

**Плейлисты создаются двумя способами:**

1. **Автоматически из подпапок.** Если в выбранной папке есть подкаталоги (например, `Music/Rock/`, `Music/Chill/`), каждый верхнего уровня становится плейлистом с именем папки. Не требует никакого ручного управления.
2. **Вручную.** На вкладке «Плейлисты» → «Создать плейлист» → имя. На карточке любого трека наведите курсор и нажмите «+» — выберите, в какой плейлист добавить.

Поиск работает по названию, исполнителю и альбому.

### Радио

При первом запуске список станций загружается из `public/radio.json`. Дальше всё сохраняется в IndexedDB и редактируется через UI.

**Добавление станции:** «+ Добавить радиостанцию» → название, URL потока, опционально иконка.

**Нюансы:**

- HTTP-потоки на HTTPS-страницах блокируются браузером (mixed content). Если приложение открыто по HTTPS, поток должен быть тоже HTTPS — иначе понадобится proxy (см. ниже).
- ICY-метаданные («сейчас играет») браузерам недоступны из JavaScript. Если нужно — поднимайте proxy.

### Аудиокниги

При добавлении книги доступны два источника:

- **Папка** (Chrome / Edge / Opera) — выбираете каталог через File System Access API; permission запоминается и переживает перезапуск.
- **ZIP-архив** — сам zip-файл сохраняется в IndexedDB, распаковывается по запросу через `jszip`.

Внутри книги:

- Главы автоматически сортируются по имени файла (natural sort, поэтому `chapter2.mp3` идёт перед `chapter10.mp3`).
- Обложка ищется в порядке: `cover.jpg/png/webp` → любое изображение в источнике → ID3-тег первого mp3.
- Активная глава подсвечивается, рядом — анимированный эквалайзер.
- Встроенный плеер на странице книги: ±30 секунд, прев/нэкст глава, прогресс-бар, селектор скорости (0.75× / 1× / 1.25× / 1.5× / 1.75× / 2×).
- Глобальный нижний Player Bar в этом режиме скрыт, чтобы не перекрывать главы.

### ТВ

При первом запуске загружается `public/IPTVstable.m3u8`.

**Добавление плейлиста:** «+ Добавить M3U» — поддерживается:

- загрузка `.m3u` / `.m3u8` файла,
- прямая ссылка на плейлист.

Каналы сгруппированы по `group-title` из `#EXTINF`, отображаются с логотипами (`tvg-logo`).

**Просмотр канала** открывает отдельную страницу: большой плеер слева, список каналов справа (поиск + фильтр по категории). Клик по соседнему каналу переключает поток без выхода из view.

Движок выбирается автоматически:

- `.m3u8` → hls.js (или нативный HLS в Safari)
- `.ts` → mpegts.js
- остальное → нативный `<video>`

## Совместимость браузеров

| Возможность                      | Chrome / Edge / Opera | Firefox    | Safari            |
| -------------------------------- | --------------------- | ---------- | ----------------- |
| File System Access («Папка»)     | ✅                    | ❌         | ❌                |
| File API («Добавить файлы»)      | ✅                    | ✅         | ✅                |
| HLS через hls.js                 | ✅                    | ✅         | ✅ (или нативно)  |
| MPEG-TS через mpegts.js          | ✅                    | ✅         | ⚠️ MSE-зависимо  |
| DASH (`.mpd`)                    | ⚠️ нативно           | ❌         | ❌                |

В Firefox / Safari «Добавить папку» не отображается — доступно только «Добавить файлы».

## Опциональный proxy для HTTP-радио и ICY-метаданных

Если нужны HTTP-only потоки на HTTPS-сайте или title «сейчас играет» — поднимите тонкий proxy:

- `/proxy/stream?url=...` — проксирует поток, добавляет CORS-заголовки, апгрейдит http → https.
- `/proxy/icy?url=...` — читает ICY-метаданные, отдаёт `{ title, artist }`.

В текущем MVP не реализован, чтобы не тащить backend без явной необходимости.

## Известные ограничения

- Локальные файлы, добавленные через `<input type="file">`, не сохраняются между сессиями — нужно переоткрывать. File System Access API (Chrome / Edge / Opera) лишён этого ограничения.
- ID треков привязаны к относительному пути файла. Перенос файлов между папками разорвёт связь с ручными плейлистами.
- Видеопоток IPTV не управляется из нижнего Player Bar — это намеренно: бар обслуживает только аудио, ТВ открывается в отдельной странице.
