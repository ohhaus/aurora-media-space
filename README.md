# Aurora

Локальный медиаплеер с интерфейсом Liquid Glass: музыка, радио, IPTV, видео и аудиокниги.

- **Моя музыка** — локальная библиотека (треки + плейлисты, сортировка, два вида списка)
- **Радио** — интернет-радио (aacp / AAC+ / mp3), редактирование станций, своя иконка
- **ТВ** — IPTV по M3U-плейлистам (HLS / MPEG-TS / DASH), bundled-плейлисты из `public/iptv/`
- **Видео** — личная видеотека (mp4 / webm / mkv / mov / m4v / avi / ogv), тот же плеер, что в ТВ
- **Аудиокниги** — папка или ZIP с mp3-главами, переименование и обложка, скорость, навигация по главам
- **Хранилище** — обзор IndexedDB и частичный / полный сброс

Переключение режимов не прерывает воспроизведение — Player Bar внизу общий (скрыт только в режимах ТВ / Видео / Аудиокниги, где работает встроенный видеоплеер).

## Стек

- React 18 + Vite + TypeScript
- Tailwind CSS (тёмная Liquid-Glass-тема, акцент `#9b8cff`)
- Zustand — единый стор плеера
- IndexedDB (`idb-keyval`) — станции, плейлисты, file-handles, blobs (zip-книг, обложек, видео)
- `music-metadata` — теги и обложки треков
- `hls.js` — HLS-потоки
- `mpegts.js` — сырые MPEG-TS потоки
- `jszip` — распаковка zip-архивов с аудиокнигами и музыкой

Никакого backend и БД — всё хранится в браузере.

## Запуск через Docker (рекомендуется)

Требуется только Docker + Docker Compose. Node.js и npm на машине не нужны.

```bash
docker compose up -d --build
```

Откройте <http://localhost:8080>.

Остановить:

```bash
docker compose down
```

Multi-stage Dockerfile собирает приложение внутри контейнера (Node 20-alpine), затем раздаёт статику через nginx — итоговый образ ~50 MB.


## Запуск локально (без Docker)

Нужен Node.js ≥ 20.

```bash
npm install
npm run dev      # dev-режим → http://localhost:5173
npm run build    # production-бандл в dist/
npm run preview  # локальный просмотр продакшен-бандла
```

## Использование

### Моя музыка

Две вкладки: **Треки** и **Плейлисты**.

**Добавление музыки:**

- **«Добавить папку»** (Chrome / Edge / Opera) — выбираете корневую папку через File System Access API. Приложение рекурсивно сканирует её, читает теги и обложки. Permission запоминается в IndexedDB — при следующем запуске нужно лишь подтвердить доступ, выбирать папку заново не придётся.
- **«Добавить файлы»** — выбор отдельных файлов; работает во всех браузерах (для Firefox / Safari это единственный путь).
- **«Добавить ZIP»** — `.zip` с аудиофайлами целиком сохраняется в IndexedDB, треки распаковываются по запросу. Каждый ZIP автоматически становится отдельным плейлистом. macOS-мусор (`._<имя>` и `__MACOSX/`) отфильтровывается на лету, дубликатов не будет.

**Сортировка и вид (на вкладке «Треки»):**

- сортировка: недавно добавленные / по имени А-Я / по имени Я-А / по исполнителю,
- переключатель **сетка ↔ список** (списковый вид — как в плейлистах).

**Плейлисты создаются тремя способами:**

1. **Автоматически из подпапок.** Если в выбранной папке есть подкаталоги (например, `Music/Rock/`, `Music/Chill/`), каждый верхнего уровня становится плейлистом с именем папки.
2. **Автоматически из ZIP.** Любой добавленный ZIP-архив превращается в плейлист — удаление его с вкладки «Плейлисты» удаляет и сам архив.
3. **Вручную.** На вкладке «Плейлисты» → «Создать плейлист» → имя. Внутри плейлиста — «+ Добавить треки» с галочками. На карточке любого трека в гриде есть «+» для быстрого добавления.

Удаление трека, плейлиста или ZIP-архива открывает кастомное окно подтверждения в стиле Liquid Glass — никаких системных `confirm()`. Поиск работает по названию, исполнителю и альбому.

### Радио

При первом запуске список станций загружается из `public/radio.json`. Дальше всё сохраняется в IndexedDB и редактируется через UI.

- **«+ Добавить радиостанцию»** — название, URL потока, иконка (по URL или локальный файл).
- На карточке станции при наведении появляются **карандаш** (редактирование названия, URL и иконки) и **корзина** (с подтверждением). Загруженная локально иконка хранится как blob прямо в IndexedDB и переживает перезапуск.

**Нюансы:**

- HTTP-потоки на HTTPS-страницах блокируются браузером (mixed content). Если приложение открыто по HTTPS, поток должен быть тоже HTTPS — иначе понадобится proxy (см. ниже).
- ICY-метаданные («сейчас играет») браузерам недоступны из JavaScript. Если нужно — поднимайте proxy.

### Аудиокниги

При добавлении книги доступны два источника:

- **Папка** (Chrome / Edge / Opera) — выбираете каталог через File System Access API; permission запоминается и переживает перезапуск.
- **ZIP-архив** — сам zip-файл сохраняется в IndexedDB, распаковывается по запросу через `jszip`. macOS-мусор (`._...`, `__MACOSX/`) пропускается.

На карточке книги — **карандаш** (переименование + загрузка своей обложки) и **корзина** с кастомным подтверждением. Своя обложка хранится в IndexedDB как blob и имеет приоритет над `cover.*` из источника.

Внутри книги:

- Главы автоматически сортируются по имени файла (natural sort, поэтому `chapter2.mp3` идёт перед `chapter10.mp3`).
- Обложка ищется в порядке: своя → `cover.jpg/png/webp` в источнике → любое изображение → ID3-тег первого mp3.
- Активная глава подсвечивается, рядом — анимированный эквалайзер.
- Встроенный плеер на странице книги: ±30 секунд, прев/нэкст глава, прогресс-бар, селектор скорости (0.75× / 1× / 1.25× / 1.5× / 1.75× / 2×).
- Глобальный нижний Player Bar в этом режиме скрыт, чтобы не перекрывать главы.

### ТВ

Плейлисты подгружаются автоматически при каждом запуске из `public/iptv/`. Их можно дополнить временными плейлистами через UI.

#### Плейлисты из `public/iptv/`

Положите ваши `.m3u` / `.m3u8` файлы в `public/iptv/` и пропишите их в `public/iptv/index.json`:

```json
[
  { "name": "IPTV (стабильные)", "file": "IPTVstable.m3u8" },
  { "name": "IPTV RU",           "file": "IPTVru.m3u" }
]
```

При каждом запуске Aurora перечитывает индекс и сами файлы с `cache: 'no-store'` — отредактировали файл, перезагрузили вкладку, новые каналы появились. Никаких сбросов IndexedDB. В UI такие плейлисты помечены ярлыком **Bundled** и не удаляются (они приходят из репозитория).

#### Добавление через UI

**«+ Добавить M3U»** — поддерживается:

- загрузка `.m3u` / `.m3u8` файла,
- прямая ссылка на плейлист.

Такие плейлисты сохраняются в IndexedDB и удаляются крестиком с кастомным подтверждением.

Каналы сгруппированы по `group-title` из `#EXTINF`, отображаются с логотипами (`tvg-logo`).

**Просмотр канала** открывает отдельную страницу: большой плеер слева, правая панель — поиск, горизонтальный chip-рейл категорий (с количеством каналов) и список каналов с логотипами. Клик по соседнему каналу переключает поток без выхода из view. **F** — фуллскрин (игнорируется в полях ввода). В шапке — кнопки переключения каналов по порядку и счётчик N / M.

Движок выбирается автоматически:

- `.m3u8` → hls.js (или нативный HLS в Safari)
- `.ts` → mpegts.js
- остальное → нативный `<video>`

### Видео

Личная видеотека из локальных файлов. Используется тот же плеер, что в ТВ.

- **«Добавить видео»** — выбор одного или нескольких файлов (`mp4`, `webm`, `mkv`, `mov`, `m4v`, `avi`, `ogv`). В Chrome / Edge / Opera сохраняется handle файла на диске; в остальных браузерах — blob внутри IndexedDB.
- На карточке — **карандаш** (название + свой постер) и **корзина** с подтверждением.
- При воспроизведении доступен **F** для фуллскрина и тот же набор контролов, что в ТВ.
- Декодинг идёт через нативные кодеки браузера — экзотические потоки в mkv / avi могут не воспроизвестись.

### Хранилище

Раздел «Хранилище» показывает, что лежит в IndexedDB, и позволяет очистить:

- только музыку (handles, ZIP-архивы, плейлисты, скрытые треки),
- только аудиокниги,
- только видео,
- или всё разом («Полный сброс Aurora»).

Каждое действие требует подтверждения в кастомной модалке — никаких системных `confirm()`. Исходные файлы на диске не трогаются.

## Совместимость браузеров

| Возможность                      | Chrome / Edge / Opera | Firefox      | Safari            |
| -------------------------------- | --------------------- | ------------ | ----------------- |
| File System Access («Папка»)     | ✅                    | ❌           | ❌                |
| File API («Добавить файлы»)      | ✅                    | ✅           | ✅                |
| ZIP-источник (аудио / книги)     | ✅                    | ✅           | ✅                |
| HLS через hls.js                 | ✅                    | ✅           | ✅ (или нативно)  |
| MPEG-TS через mpegts.js          | ✅                    | ✅           | ⚠️ MSE-зависимо  |
| DASH (`.mpd`)                    | ⚠️ нативно           | ❌           | ❌                |
| Видео mp4 / webm                 | ✅                    | ✅           | ✅                |
| Видео mkv / avi                  | ⚠️ зависит от кодеков| ⚠️           | ⚠️                |

В Firefox / Safari «Добавить папку» не отображается — доступно только «Добавить файлы».

## Опциональный proxy для HTTP-радио и ICY-метаданных

Если нужны HTTP-only потоки на HTTPS-сайте или title «сейчас играет» — поднимите тонкий proxy:

- `/proxy/stream?url=...` — проксирует поток, добавляет CORS-заголовки, апгрейдит http → https.
- `/proxy/icy?url=...` — читает ICY-метаданные, отдаёт `{ title, artist }`.

В текущем MVP не реализован, чтобы не тащить backend без явной необходимости.

## Оптимизация памяти

- Object-URL для обложек треков отзываются при пересборе библиотеки и при unmount.
- Режим «Видео» монтируется только когда активен — `<video>`-декодер не висит фоном.
- Превью-URL иконок радио и постеров видео отзываются при обновлении.

## Известные ограничения

- Локальные файлы, добавленные через `<input type="file">`, не сохраняются между сессиями — нужно переоткрывать. File System Access API (Chrome / Edge / Opera) лишён этого ограничения.
- ID треков привязаны к относительному пути файла. Перенос файлов между папками разорвёт связь с ручными плейлистами.
- Видеопоток ТВ / Видео не управляется из нижнего Player Bar — это намеренно: бар обслуживает только аудио, видео открывается в отдельной странице.
- mkv / avi с экзотическими кодеками не воспроизведутся напрямую — браузер использует только нативные декодеры платформы.
